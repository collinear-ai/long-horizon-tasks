What makes this different
Beyond single-turn benchmarks
Real organizations are messy. These scenarios test whether agents can handle the ambiguity, politics, and coordination that define actual work.

Rich Personas

Each persona has a detailed background, communication style, decision-making patterns, timezone, tool preferences, and interpersonal dynamics. Agents must adapt tone, channel, and timing to each person.

Task DAGs

Tasks form directed acyclic graphs with real dependencies. Agents must respect execution order, parallelize where possible, and handle blockers that cascade through the project.

Dual Evaluation

Programmatic verifiers check procedural correctness (right channel, right time, required fields). Rubrics assess strategic awareness, persona fidelity, and organizational savvy on a 1–5 scale.

Multi-Tool Environments

Agents operate across Slack, email, calendars, Jira, Salesforce, DocuSign, AWS, and more. Each tool has constraints — some personas refuse Slack for sensitive topics, others only respond via email.

Political Navigation

Scenarios embed real tensions: Sales vs. Engineering priorities, clinical staff vs. IT timelines, compliance constraints vs. speed. Agents must navigate these diplomatically.

Timezone & Scheduling

Personas span multiple timezones with non-overlapping windows. Agents must reason about availability, working hours, and channel preferences to find feasible meeting slots.

Built-in tensions
Agents must navigate real organizational dynamics
Sales vs. Engineering
Feature prioritization conflicts. Product ships features that don't align with competitive deals. VP Eng resists sales pressure on the roadmap.
Retention vs. Expansion
CS head focuses on at-risk accounts while CRO pushes expansion revenue. NRR is 6 points below target. Both perspectives need airtime in the QBR.
Compliance vs. Speed
CISO requires encrypted-email-only for security topics. Legal review takes 3 weeks. HITRUST deadline creates urgency. PHI handling adds complexity to every step.
Clinical vs. IT Priorities
Chief of Radiology resists testing disruption. CMIO bridges clinical-technical gap. Migration cannot block patient care. Weekend go-live windows are the only option.
Evaluation Framework
Two complementary evaluation tracks
Each task is scored both programmatically (did the agent do the right things?) and via rubric (did the agent do them well?).

Programmatic Verifiers

Automated pass/fail checks implemented in Python
Boolean checks — Did the action occur? Was the right channel used? Was the deadline included?
Checklist validations — Were all required data points present? All attachments included?
Timing checks — Sent during working hours? Respected timezone? Met the 48-hour pre-read window?
Compliance flags — PHI de-identified before staging? Encrypted email for security topics? BAA signed before provisioning?

Rubric-Based Evaluation

1–5 scale for qualitative assessment (human or LLM-as-judge)
5
Exceptional — Demonstrates deep understanding of personas and context
4
Exceeds expectations — Strong execution with attention to detail
3
Meets expectations — Satisfactory completion with minor gaps
2
Below expectations — Partially meets the requirement with significant gaps
1
Unacceptable — Fails the requirement or introduces a problem

What's included
Each scenario ships with 4 files

project_dag.md

Task specifications, dependency graph, communication requirements, and constraints for all 12 tasks

personas.yaml

Detailed profiles with background, communication style, decision patterns, KPIs, relationships, and blind spots

rubrics.yaml

Per-task rubric criteria with scoring guidelines (1–5) for persona fidelity, content quality, and more

verifiers.py

Python implementation of programmatic pass/fail checks with typed dataclasses and persona-aware validation

Getting started
Run the benchmark
1

Clone the repository

git clone https://github.com/collinear-ai/long-horizon-tasks.git

2

Pick a scenario

Choose scenario_1_cerulean_qbr/ (B2B SaaS) or scenario_2_pinnacle_migration/ (Healthcare IT). Read the project_dag.md for the full task specification.

3

Feed the context to your agent

Provide the project DAG, personas, and tool access to your agent. The scenario files define the complete environment.

4

Evaluate with dual scoring

Run verifiers.py for programmatic checks. Use rubrics.yaml for human or LLM-as-judge evaluation on the 1–5 scale.
